Machine Heart

Machine Heart is a 2.5D Sci-fi Puzzle-Adventure Game featuring two interconnected storylines.

Story

You'll play as Avis, a defiant scientist hunted for her research, and Vox, a sentient robot who lost all his memory. By bringing together two storylines from two distinct eras, you'll uncover the striking secret behind the fate of humanity.

Throughout your journey, you will learn to solve challenging cross-timeline puzzles with a mix of stealth, bike chasing, and exploration. Will you be able to find out what truly happened to all of mankind?

My Contributions

  1. Dialogue System
    • Display the text in the dialogue box with a typing effect to enhance visual effects
    • Response box displays multiple available responses for the player to select which trigger specific events or dialogues on selection
    • Easy and flexible creation of dialogues from the editor itself using a scriptable object in Unity to enable monologues and conversations between two characters
  2. Voiceover System
    • Added voiceover to easily implement voice over by just adding the voice clip to the scriptable object from the editor itself
    • Integrated the Voiceover in the Dialogue System to reduce development overhead by reducing coding time and increase efficiency of the overall development process
  3. Save/Load System
    • Designed and implemented a Save System for the game that stores the data and attributes in JSON format using serializable objects
    • Implemented features accessible while editing to control the save system behavior reduce development overhead by reducing coding time
    • The save system saves the player data, location, stats, level information and the world information enabling the player to save and come back to the game thus improving player experience
  4. Enemy AI
    • Implemented an AI System to control and manage enemy behaviour by deciding spawn position at runtime based on player position and behavior
  5. Level Design
    • Contributed to Level Design by debugging the interactions of the player with their environment
  6. Audio Manager
    • Implemented an Audio Manager System that enabled the developers and level designers to add sound effects to the game seamlessly from the editor without writing additional code
    • The Audio Manager handled the events that trigger the playing of the sound effect and played the sound effects appropriately
